# Trading Database System

## Overview
This project involves designing a relational database system for a trading company to manage operations efficiently.

## Objectives
- Design a structured database system
- Model relationships between entities
- Support business operations with SQL queries

## Key Components
- Entity-Relationship (ER) Diagram
- Database schema design
- Normalization of data
- SQL queries (JOIN, filtering, etc.)

## Features
- Trade management system
- Customer and contract tracking
- Risk assessment integration
- Data retrieval using SQL queries

## Skills Demonstrated
- Database design
- SQL query development
- Data modeling and normalization
